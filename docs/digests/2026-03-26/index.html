Operating System Design✓ reviewed

Tech · Systems  ·  ~4 min read

Core Concept

An operating system is privileged system software that mediates between application code and physical hardware, providing resource abstractions, isolation guarantees, and common services. Its design space is governed by a fundamental tension: efficiency (minimizing indirection) versus safety (enforcing protection boundaries).

The OS executes in a privileged execution mode (kernel mode / ring 0), while user applications run in unprivileged mode. Transitions between these modes occur via system calls (syscalls) — synchronous software-triggered traps — or interrupts — asynchronous hardware signals.

INFO
A syscall is not a function call. It triggers a mode switch through a gate (e.g., SYSCALL/SYSRET on x86-64), saving registers, switching stacks, and jumping to a kernel handler. Round-trip cost is typically 100–300 ns on modern hardware, often dominating I/O latency in fine-grained workloads.

The core abstractions an OS must provide:

| Abstraction | Underlying Resource |

|---|---|

| Process / Thread | CPU |

| Virtual Memory | Physical RAM + swap |

| File / inode | Persistent storage |

| Socket / pipe | Network / IPC |

| Device file | Peripheral hardware |

Why It Matters

OS design choices propagate through the entire software stack. Scheduler policy determines tail latency for cloud services. Memory allocator design shapes GC pause times. Filesystem semantics define the failure modes of databases.

$$\text{Throughput} \approx \frac{N}{\text{Latency}} \quad \text{(Little's Law)}$$

where $N$ is concurrency. The OS controls both $N$ (thread scheduling, connection limits) and latency (syscall overhead, lock contention), making it the primary determinant of system-level performance.

Beyond performance: OS design is the foundation of security. The kernel's trusted computing base (TCB) must be minimal — every line of kernel code is a potential privilege-escalation vector. This motivates microkernel architectures (Mach, seL4) despite their IPC overhead.

TIP
The seL4 microkernel is formally verified correct in Isabelle/HOL — its implementation provably matches its specification. This is the gold standard for TCB minimization: ~10K lines of C, fully proved. Compare to Linux at ~30M lines.

Key Details

1. Scheduling Algorithms and Their Trade-offs

Classical schedulers (Round Robin, MLFQ) optimize for average throughput. Real-time schedulers (EDF, Rate-Monotonic) provide deadline guarantees. The Completely Fair Scheduler (CFS) in Linux uses a red-black tree keyed on virtual runtime to approximate fair CPU sharing in $O(\log n)$ per operation.

2. Virtual Memory and the TLB

The MMU walks a multi-level page table (4-level on x86-64) to translate virtual → physical addresses. The TLB caches recent translations. A TLB miss costs ~30–100 cycles; a page fault (OS intervention required) costs thousands. Huge pages (2 MB, 1 GB) reduce TLB pressure at the cost of internal fragmentation.

3. Monolithic vs. Microkernel vs. Exokernel

  • Monolithic (Linux, XNU): all subsystems in one address space — fast IPC, large TCB.
  • Microkernel (seL4, QNX): minimal kernel, services as user processes — isolated but IPC-heavy.
  • Exokernel (MIT): kernel only enforces protection; library OSes implement abstractions — maximum flexibility, minimal abstraction overhead.

4. Synchronization Primitives

Spinlocks waste CPU on contention; mutexes sleep but involve kernel transitions. The futex (fast userspace mutex) in Linux uses an atomic compare-and-swap in userspace and only drops into the kernel on contention — the common uncontended path costs ~2 ns.

5. Filesystem Consistency

Journaling (ext4, NTFS), copy-on-write (ZFS, Btrfs), and log-structured (F2FS, NILFS) are the three dominant strategies for crash consistency. COW provides snapshots and checksumming "for free" but amplifies write I/O; journaling has lower write amplification but no inherent data integrity checks.

Current State

Active research fronts include:

  • Unikernels (MirageOS, Unikraft): compile only necessary OS components into a single-address-space image for VMs, eliminating syscall overhead entirely for cloud-native workloads.
  • io_uring (Linux 5.1+): an async I/O interface using shared-memory ring buffers, allowing thousands of I/O operations to be submitted/reaped with zero syscalls in the steady state.
  • eBPF: allows verified user-supplied bytecode to run in kernel context, enabling programmable networking, tracing, and security policy without kernel patches — effectively a safe, JIT-compiled extension mechanism.
  • RISC-V and capability hardware (CHERI): hardware-enforced pointer capabilities that could fundamentally change how kernels enforce memory isolation, potentially making type-safe OS components trivially enforceable.
  • AI/ML workload scheduling: modern datacenter OSes must co-schedule GPU kernels, handle NUMA topology across accelerators, and manage memory bandwidth — problems ill-addressed by classical scheduler theory.

Monte Carlo Methods✓ reviewed

Math · Statistics  ·  ~4 min read

Core Concept

Monte Carlo methods are a family of algorithms that estimate deterministic quantities by drawing repeated random samples and aggregating results. The unifying idea: replace analytic intractability with stochastic approximation, exploiting the Law of Large Numbers to guarantee convergence.

INFO: Formal statement
For a function $g: \mathbb{R}^d \to \mathbb{R}$, Monte Carlo estimates an expectation $$I = \mathbb{E}_{X \sim p}[g(X)] = \int g(x)\, p(x)\, dx$$ by the sample mean $\hat{I}_N = \frac{1}{N} \sum_{i=1}^{N} g(X_i)$, with $X_i \overset{\text{iid}}{\sim} p$. By the CLT, the error scales as $\mathcal{O}(N^{-1/2})$, independent of dimension $d$.

This dimension-independence is the core reason Monte Carlo dominates numerical integration in high dimensions — classical quadrature degrades exponentially in $d$ (the curse of dimensionality), while MC does not.

Why It Matters

The $\mathcal{O}(N^{-1/2})$ rate is deceptively powerful. For a 100-dimensional integral, any deterministic grid method requires exponentially many points; Monte Carlo requires the same $N$ regardless. This makes it the default tool in:

  • Bayesian inference — posterior sampling via MCMC when closed forms are unavailable
  • Financial engineering — option pricing, risk metrics (VaR, CVaR) under stochastic models
  • Particle physics & nuclear engineering — radiation transport, reactor safety assessments
  • Reinforcement learning — policy evaluation via rollouts; Monte Carlo Tree Search (MCTS)
  • Diffusion models / generative AI — sampling from score-based SDEs
TIP: Why randomness solves *deterministic* problems
Determinism lives in the quantity being estimated (e.g. a definite integral). Randomness is the sampling strategy. The method probabilistically brackets the truth — you trade exact answers for scalable approximations with known error statistics.

Key Details

1. Variance reduction is the central engineering problem.

Raw MC has high variance. Techniques like importance sampling, control variates, stratified sampling, and quasi-Monte Carlo (using low-discrepancy sequences such as Sobol or Halton) can dramatically reduce estimator variance without increasing $N$.

2. MCMC extends MC to unnormalized distributions.

When sampling directly from $p$ is impossible, Markov Chain Monte Carlo (Metropolis-Hastings, HMC, NUTS) constructs a chain whose stationary distribution is $p$. Convergence diagnostics (Gelman-Rubin $\hat{R}$, ESS) become critical.

3. The multilevel paradigm breaks the cost-accuracy tradeoff.

Multilevel Monte Carlo (MLMC) exploits a hierarchy of approximations at different resolutions. The key identity:

$$\mathbb{E}[g(X^L)] = \mathbb{E}[g(X^0)] + \sum_{\ell=1}^{L} \mathbb{E}[g(X^\ell) - g(X^{\ell-1})]$$

Cheap coarse levels handle bulk variance; expensive fine levels only correct residuals. This can reduce cost from $\mathcal{O}(\varepsilon^{-3})$ to $\mathcal{O}(\varepsilon^{-2})$ for SDEs.

4. Quasi-Monte Carlo achieves faster convergence but with caveats.

QMC sequences replace pseudo-random draws with deterministic low-discrepancy sequences, achieving $\mathcal{O}((\log N)^d / N)$ error — better than MC's $\mathcal{O}(N^{-1/2})$ for smooth integrands. But the $(\log N)^d$ prefactor bites in high $d$, and error bounds require smoothness conditions often violated in practice.

5. Sequential Monte Carlo (particle filters) handles dynamic systems.

SMC propagates a weighted particle population through time, enabling online Bayesian filtering for state-space models — the backbone of SLAM, radar tracking, and Bayesian sequential inference.

Group Theory✓ reviewed

Math · Algebra  ·  ~4 min read

Core Concept

INFO: Definition
A group is a set $G$ together with a binary operation $\cdot : G \times G \to G$ satisfying four axioms: 1. Closure: $a, b \in G \Rightarrow a \cdot b \in G$ 2. Associativity: $(a \cdot b) \cdot c = a \cdot (b \cdot c)$ 3. Identity: $\exists\, e \in G$ such that $e \cdot a = a \cdot e = a$ 4. Inverses: $\forall\, a \in G,\, \exists\, a^{-1}$ such that $a \cdot a^{-1} = e$

If additionally $a \cdot b = b \cdot a$ for all $a, b$, the group is abelian. The order $|G|$ is the cardinality of $G$.

A subgroup $H \leq G$ is a subset closed under the group operation and inverses. The index $[G:H] = |G|/|H|$ (Lagrange's theorem) — every subgroup's order divides the group's order.

Key structural vocabulary: normal subgroups ($N \trianglelefteq G$, where $gNg^{-1} = N$) enable quotient groups $G/N$; homomorphisms $\phi: G \to H$ preserve group structure; the kernel $\ker\phi = \{g \mid \phi(g) = e_H\}$ is always normal. The First Isomorphism Theorem states $G/\ker\phi \cong \mathrm{im}\,\phi$.

Why It Matters

Group theory is the mathematical backbone of symmetry. Any physical transformation that can be undone and composed forms a group — this single observation unifies vast swaths of physics and chemistry.

  • Particle physics: The Standard Model is built on the gauge group $SU(3) \times SU(2) \times U(1)$. Representation theory dictates which particles can exist and how they interact. Three of the four fundamental forces arise from local symmetry principles.
  • Crystallography: The 230 space groups exhaustively classify all possible crystal symmetries — a finite classification with infinite material consequences.
  • Cryptography: Discrete logarithm hardness in groups like $\mathbb{Z}_p^*$ or elliptic curve groups underpins RSA, Diffie-Hellman, and ECDSA.
  • Quantum mechanics: Wigner's theorem guarantees symmetry operations act as (anti-)unitary operators; the representation theory of $SU(2)$ produces the quantization of angular momentum and spin.
TIP: Algebraic Universality
Rings, fields, and vector spaces are all groups with extra structure. A ring is an abelian group $(R, +)$ with a compatible multiplication. This means group-theoretic results propagate upward through the entire edifice of algebra.

Key Details

1. Simple groups and Jordan–Hölder. Every finite group has a composition series with simple group factors, and those factors are unique up to permutation (Jordan–Hölder theorem). Simple groups are the "atoms" — irreducible building blocks.

2. Sylow theorems. For a finite group $|G| = p^k m$ with $\gcd(p,m)=1$, Sylow $p$-subgroups of order $p^k$ exist, are conjugate, and their count $n_p$ satisfies $n_p \equiv 1 \pmod{p}$ and $n_p \mid m$. This is a primary tool for classifying groups of small order.

3. Free groups and presentations. Any group can be presented as $\langle S \mid R \rangle$ — generators modulo relations. The free group $F_n$ on $n$ generators is the "most general" group with those generators; every group with $n$ generators is a quotient of $F_n$.

4. Lie groups as continuous symmetries. A Lie group is simultaneously a smooth manifold and a group. Its local structure is captured by the Lie algebra $\mathfrak{g} = T_e G$, making infinitesimal analysis tractable. The exponential map $\exp: \mathfrak{g} \to G$ connects them: $$g(\epsilon) \approx e + \epsilon X, \quad X \in \mathfrak{g}$$

5. Representation theory. A representation $\rho: G \to GL(V)$ realises group elements as linear transformations. For finite groups, Maschke's theorem guarantees complete reducibility over $\mathbb{C}$, and character theory (the trace of $\rho(g)$) fully classifies irreducible representations.

Current State

WARNING: Note: this information may be dated

The Classification of Finite Simple Groups (CFSG) — completed circa 2004 after ~10,000 journal pages — is one of mathematics' largest collaborative proofs. It identifies exactly 18 infinite families (cyclic, alternating, 16 families of groups of Lie type) plus 26 sporadic groups, the largest being the Monster group $\mathbb{M}$ of order $\approx 8 \times 10^{53}$. A "second-generation" proof reducing the page count remains an active program.

Geometric group theory (studying groups via their action on metric spaces, Cayley graphs, buildings) is highly active. Connections to low-dimensional topology — through mapping class groups, braid groups, and 3-manifold invariants — continue to generate new results.

In physics, symmetry-breaking patterns in periodically-driven (Floquet) systems and fractionalized phases are an active frontier, where the interplay of group-theoretic order parameters with non-equilibrium dynamics produces rich phase diagrams not captured by classical Landau theory.

Topology✓ reviewed

Math · Geometry  ·  ~4 min read

Core Concept

Topology studies properties of spaces preserved under continuous deformations — stretching, twisting, bending — but not tearing, gluing, or self-intersection. Informally, a coffee mug and a donut are "the same" topologically (both have genus 1), while a sphere and a torus are not.

INFO: Topological Space
A topological space is a pair $(X, \mathcal{T})$ where $X$ is a set and $\mathcal{T} \subseteq \mathcal{P}(X)$ is a topology: a collection of "open" sets satisfying: 1. $\emptyset, X \in \mathcal{T}$ 2. Arbitrary unions of elements of $\mathcal{T}$ are in $\mathcal{T}$ 3. Finite intersections of elements of $\mathcal{T}$ are in $\mathcal{T}$

The canonical morphisms are homeomorphisms — bijections $f: X \to Y$ where both $f$ and $f^{-1}$ are continuous — and homotopies, continuous deformations between maps $f, g: X \to Y$. A topological invariant is any property preserved by homeomorphism: dimension, compactness, connectedness, fundamental group, homology groups, etc.

Why It Matters

Topology is the lingua franca of modern mathematics. It underpins:

  • Analysis: the definitions of continuity, convergence, and compactness generalize seamlessly from $\mathbb{R}^n$ to arbitrary spaces.
  • Differential geometry / physics: manifolds are topological spaces with extra structure; the topology of spacetime constrains physical theories (e.g., Chern–Simons theory, topological insulators).
  • Data science: Persistent Homology and Topological Data Analysis (TDA) extract shape features from point clouds invariant to noise.
  • Computer science: domain theory (denotational semantics) uses non-Hausdorff topologies; topological methods appear in distributed computing impossibility proofs.
  • Robotics / vision: configuration spaces of robots are manifolds; surgical reconstruction systems must handle "fixed-topology" constraints on deformable tissue (an active pain point, per recent work).
TIP: The Real Power Is Abstraction
By stripping geometry down to open sets, topology isolates why theorems like the intermediate value theorem or the extreme value theorem are true — they follow from connectedness and compactness, not from the real number line specifically.

Key Details

1. Separation Axioms (T₀–T₄ hierarchy)

Not all topological spaces are equally "nice." Hausdorff spaces ($T_2$) guarantee limits are unique. Normal spaces ($T_4$) admit Urysohn functions, enabling Tietze extension. Much of classical analysis assumes at least $T_3$ (regular + $T_1$).

2. Compactness is the right generalization of "finiteness"

A space is compact iff every open cover has a finite subcover. In metric spaces this coincides with sequential compactness. Key consequence: continuous image of compact is compact, yielding the extreme value theorem in full generality.

3. Fundamental Group and Higher Homotopy

$$\pi_1(X, x_0) = \{\text{loops at } x_0\} / \text{homotopy rel. endpoints}$$

This group captures "1-dimensional holes." Higher groups $\pi_n$ are harder; $\pi_n(S^n) \cong \mathbb{Z}$ but $\pi_3(S^2) \cong \mathbb{Z}$ (Hopf fibration) — a famous surprise.

4. Homology vs. Homotopy

Singular homology $H_n(X; \mathbb{Z})$ is generally more computable than homotopy groups and still distinguishes spaces by their "holes" in each dimension. The Euler Characteristic $\chi = \sum_n (-1)^n \text{rank}(H_n)$ is a coarse but powerful invariant.

5. Fixed-Point Theorems

The Brouwer Fixed-Point Theorem (every continuous $f: D^n \to D^n$ has a fixed point) and Lefschetz Fixed-Point Theorem are purely topological results with profound implications in economics, game theory, and dynamical systems.

Current State

WARNING: Note: this information may be dated

Active frontiers include:

  • Topological Data Analysis: persistent homology pipelines (Ripser, Gudhi) are now standard in ML pipelines for shape analysis and feature engineering on point clouds.
  • Applied topology in deep learning: enforcing topological constraints in neural network outputs (e.g., preserving connectivity in segmentation maps, reconstructing deformable tissue geometry in surgical robotics where fixed-topology mesh assumptions break down).
  • Floer homology and low-dimensional topology: the Poincaré conjecture (dimension 3) was resolved by Perelman via Ricci flow in 2003, but the smooth 4D Poincaré conjecture remains open.
  • Higher category theory / $\infty$-toposes: Lurie's work re-frames homotopy theory as a branch of category theory, blurring the line between topology and logic.
