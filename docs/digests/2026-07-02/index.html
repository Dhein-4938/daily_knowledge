Operating System Design✓ reviewed

Tech · Systems  ·  ~5 min read

Core Concept

An operating system is the privileged software layer that arbitrates access to hardware resources—CPU, memory, I/O devices, storage—and presents a stable, abstract interface to user-space programs via system calls.

[!info] Formal definition

The OS can be modeled as a resource multiplexer and hardware abstractor: it enforces isolation between processes, schedules CPU time, and exposes a virtual machine to each process. Formally, for $n$ competing processes $P_1, \ldots, P_n$ each demanding resource $R$, the OS defines a policy $\pi: \mathcal{P}(R) \to \text{Allocation}$ subject to safety and liveness constraints.

The central tension in OS design is the mechanism vs. policy separation: the kernel provides mechanisms (context switching, page tables, interrupt handling), while policies (scheduling discipline, eviction strategy, security rules) are ideally kept configurable or user-defined. This principle, articulated by Lampson and Levin, remains the backbone of microkernel and exokernel architectures.

Why It Matters

Operating systems are the substrate on which all other software runs. Their design choices propagate upward through the entire software stack:

  • Performance: Cache-oblivious scheduling, NUMA-aware memory allocation, and zero-copy I/O paths can alter end-to-end throughput by orders of magnitude.
  • Security posture: The OS defines the trust boundary. A kernel vulnerability collapses all isolation guarantees simultaneously.
  • Abstractions exported: The POSIX file/process/socket model shaped four decades of software design; deviating from it (e.g., Plan 9's "everything is a file, including the network namespace") changes what application patterns are natural.
  • Cross-domain relevance: Scheduler theory borrows from queueing theory; virtual memory maps directly to the theory of paging in combinatorics; IPC protocol design intersects with distributed systems consensus.

[!tip] OS design as language design

The system call API is an ABI-stable language the OS speaks to user space. Like programming languages, every OS represents a bet on which abstractions pay off over decades. Linux's monolithic kernel won partly because its ABI stability let an enormous ecosystem accumulate — a lesson in network effects over theoretical elegance.

Key Details

1. Kernel Architectures

| Architecture | Mechanism | Trade-off |

|---|---|---|

| Monolithic | All services in kernel space | Fast IPC, large TCB |

| Microkernel | Minimal kernel; services as user processes | Small TCB, IPC overhead |

| Exokernel | Expose raw hardware; library OSes | Maximum flexibility, complex apps |

| Unikernel | Single address space, one application | Near-zero overhead, no isolation |

2. Scheduling Complexity

CPU scheduling is NP-hard in the general case. Real schedulers use heuristics: Linux's Completely Fair Scheduler (CFS) tracks virtual runtime $v_i$ per process and always dispatches the process minimising $v_i$, approximating max-min fairness in $O(\log n)$ per operation via a red-black tree.

3. Virtual Memory & the TLB Bottleneck

The page table walk is $O(\log_{512} \text{VA space})$ on x86-64 (4 levels). The TLB caches translations, but TLB shootdowns under multi-core invalidation are an underappreciated scalability bottleneck. Huge pages (2 MiB, 1 GiB) trade internal fragmentation for fewer TLB misses — a classic space-time tradeoff.

4. Interrupt Handling & the Kernel/User Boundary

Every system call crosses the privilege ring boundary, flushing pipeline state. Post-Spectre/Meltdown, KPTI (Kernel Page-Table Isolation) maps separate page tables for kernel and user space, adding 10–30% syscall overhead on some workloads — a security constraint permanently reshaping performance models.

5. Synchronisation Primitives

Locks, semaphores, and condition variables are OS-mediated. Futexes (Fast Userspace muTEXes) optimise the common uncontended case by avoiding syscalls entirely — the kernel is only invoked on contention. This exemplifies a recurring OS pattern: fast path in user space, slow path via kernel.

Current State

[!warning] Note: this information may be dated

Market share figures (Android 38%, Windows 33%, iOS/iPadOS 15%, macOS 4%, Linux 1% as of late 2025) shift continuously, especially in the mobile and embedded segments.

Active research fronts include:

  • Kernel bypass / io_uring: Moving I/O submission and completion entirely to shared ring buffers between kernel and user space, eliminating syscall overhead for high-throughput storage and networking.
  • Formal verification: seL4 (a microkernel) is fully formally verified in Isabelle/HOL — proving functional correctness and information-flow security. Scaling this to production kernels remains open.
  • WASM-based OS abstractions: WebAssembly runtimes are emerging as a portable, sandboxed alternative to processes, blurring the OS/runtime boundary.
  • LLM-integrated memory management: Recent work (e.g., AutoMem, 2026) treats memory as a learnable cognitive skill for agents — an architectural idea that echoes OS virtual memory but applied to language model context windows, with the model itself learning eviction and retrieval policy.
  • RISC-V & privilege model experimentation: The open ISA enables novel privilege level designs (e.g., hypervisor extensions, PMP for physical memory protection) that were impractical to prototype on x86.

[!tip] Conceptual bridge to active matter

Recent theoretical physics (Stahl et al., 2026) shows that Brownian ratchets — systems harnessing temperature differentials to do directed work — can universally simulate nonequilibrium dynamics. This maps loosely onto OS schedulers: both convert a raw energy/time budget into directed, ordered computation from an otherwise chaotic substrate.

Monte Carlo Methods✓ reviewed

Math · Statistics  ·  ~4 min read

Core Concept

INFO
Monte Carlo methods are a class of computational algorithms that use repeated random sampling to obtain numerical results for problems that are deterministic in principle but intractable analytically. Conceived by Stanisław Ulam at Los Alamos in the 1940s, they transform stochastic machinery into a general-purpose numerical solver.

The foundational insight is the Law of Large Numbers applied as a computational tool. For an integral you cannot evaluate in closed form:

$$I = \int_\Omega f(\mathbf{x})\, d\mathbf{x} \approx \frac{|\Omega|}{N} \sum_{i=1}^{N} f(\mathbf{x}_i), \quad \mathbf{x}_i \sim \mathrm{Uniform}(\Omega)$$

The estimator converges at rate $O(N^{-1/2})$ regardless of dimension — the canonical escape from the curse of dimensionality that defeats deterministic quadrature. The standard error of the estimate is:

$$\sigma_{\hat{I}} = \frac{\sigma_f}{\sqrt{N}}$$

where $\sigma_f$ is the standard deviation of $f$ over $\Omega$. Reducing error by 10× costs 100× more samples — the fundamental trade-off.

Why It Matters

Monte Carlo occupies an unusual niche: it is simultaneously a theoretical tool (probing statistical physics), an engineering tool (radiation transport, financial option pricing, circuit simulation), and an epistemological workhorse (Bayesian posterior sampling via MCMC). Three primary use classes:

  1. Numerical integration — especially in high-dimensional spaces ($d \gg 10$) where deterministic methods fail
  2. Optimization — simulated annealing, cross-entropy, and evolutionary methods exploit stochastic search to escape local minima
  3. Sampling from non-uniform distributions — the foundation of modern Bayesian computation via Markov Chain Monte Carlo
TIP
Monte Carlo's dimension-independence ($O(N^{-1/2})$ regardless of $d$) is why it dominates in Bayesian Inference and Particle Physics Simulation — both inherently high-dimensional — while deterministic quadrature rules remain preferable for $d \leq 3$.

Key Details

1. Variance Reduction Techniques

Raw sampling is often wasteful. Key techniques: importance sampling (sample from $q(\mathbf{x}) \propto |f(\mathbf{x})|$, correct with weights $w_i = f/q$), control variates (subtract a correlated term with known expectation), stratified sampling, and antithetic variates. These can reduce $\sigma_f$ dramatically without increasing $N$.

2. Markov Chain Monte Carlo (MCMC)

When direct sampling from a target $\pi(\mathbf{x})$ is impossible, construct a Markov chain whose stationary distribution is $\pi$. Metropolis-Hastings and Hamiltonian Monte Carlo (HMC) are workhorses of modern Bayesian computation. Convergence diagnostics (Gelman-Rubin $\hat{R}$, effective sample size) are non-trivial.

3. The Bias-Variance Structure

Monte Carlo estimators are typically unbiased but high-variance. This contrasts with quasi-Monte Carlo (QMC) methods, which use low-discrepancy sequences (Sobol, Halton) to achieve $O(N^{-1}(\log N)^d)$ convergence — biased in general but empirically superior for moderate $d$.

4. Ergodicity Requirements

For MCMC to work, the chain must be irreducible (can reach any state) and aperiodic. In practice, poor mixing — chains stuck in local modes — is the dominant failure mode. Techniques like parallel tempering address multimodal targets.

5. Nested Monte Carlo

Some problems (e.g., expected information gain in Bayesian optimal experimental design) require nested expectations $\mathbb{E}[\mathbb{E}[\cdot|\mathbf{x}]]$ that demand nested sampling loops, incurring $O(N^{-1/2})$ bias from the inner estimator that outer averaging cannot remove.

Current State

WARNING
Note: this information may be dated

Active research fronts include diffusion models (score-based generative models as learned samplers), flow-based methods replacing MCMC for normalizing flows, and randomized quasi-Monte Carlo (RQMC) hybridizing QMC's low-discrepancy structure with Monte Carlo's variance estimability. In physics, many-body Brownian ratchet systems (Stahl et al., 2026) demonstrate how heat-bath stochasticity — a physical Monte Carlo analog — can universally simulate active spin dynamics, blurring the line between algorithmic and physical randomness. Separately, quantum Monte Carlo variants (variational, diffusion, auxiliary-field) remain among the most competitive methods for ab initio many-body quantum problems.

Group Theory✓ reviewed

Math · Algebra  ·  ~5 min read

Core Concept

INFO: Definition
A group is a set $G$ equipped with a binary operation $\cdot : G \times G \to G$ satisfying four axioms: 1. Closure: $\forall a, b \in G,\ a \cdot b \in G$ 2. Associativity: $(a \cdot b) \cdot c = a \cdot (b \cdot c)$ 3. Identity: $\exists e \in G$ such that $e \cdot a = a \cdot e = a$ 4. Inverses: $\forall a \in G,\ \exists a^{-1}$ such that $a \cdot a^{-1} = e$

Group theory is the systematic study of symmetry via these algebraic structures. The key insight is that symmetry operations — rotations, reflections, permutations, gauge transformations — compose associatively and admit inverses, making groups the natural language for symmetry.

Important taxonomy:

  • Abelian (commutative): $a \cdot b = b \cdot a$ for all $a, b$ (e.g., $(\mathbb{Z}, +)$)
  • Finite groups: $|G| < \infty$; order governs structure via Lagrange's theorem ($|H|$ divides $|G|$ for subgroup $H$)
  • Lie groups: smooth manifolds where the group operation is $C^\infty$ (e.g., $SO(3)$, $SU(2)$, $U(1)$)
  • Simple groups: no non-trivial normal subgroups — the "atoms" of group theory

The order-$n$ symmetric group $S_n$ acts on all finite groups via Cayley's theorem: every group embeds into some $S_n$.

Why It Matters

Group theory sits at the intersection of virtually every branch of mathematics and physics.

In mathematics: Rings, fields, and vector spaces are groups with extra structure. Galois theory — which resolves the insolvability of the quintic — translates field extensions into group theory, making group theory the hidden engine behind classical algebra.

In physics: The Standard Model of particle physics is gauge theory — a theory of principal bundles with structure groups $U(1) \times SU(2) \times SU(3)$. Noether's theorem ties every continuous symmetry group to a conserved current. Crystal symmetries are classified by 230 space groups. The hydrogen atom's accidental $SO(4)$ symmetry explains the degeneracy of energy levels:

$$E_n = -\frac{m_e e^4}{2\hbar^2 n^2}$$

where $n$ labels representations of $SO(4)$, not just $SO(3)$.

In cryptography: Elliptic curve cryptography operates over the group of points on an elliptic curve over $\mathbb{F}_p$. Discrete log hardness in these groups is the security foundation of most public-key infrastructure.

TIP: Representation theory bridge
Groups are most powerful when they act on vector spaces. A representation $\rho: G \to GL(V)$ linearizes group theory, making Fourier analysis, quantum mechanics, and the classification of particles all special cases of the same framework.

Key Details

  1. Homomorphisms and kernels: A map $\phi: G \to H$ preserving the operation has $\ker\phi \trianglelefteq G$ (normal subgroup). The First Isomorphism Theorem gives $G/\ker\phi \cong \text{im}(\phi)$ — structure-preserving maps reveal internal quotient structure.
  1. Sylow theorems: For $|G| = p^k m$ with $\gcd(p,m)=1$, Sylow $p$-subgroups of order $p^k$ exist, are conjugate to each other, and their count divides $m$ while being $\equiv 1 \pmod{p}$. Essential for classifying groups of small order.
  1. Simple group classification: The Classification of Finite Simple Groups (CFSG) — completed ~2004 across $>10{,}000$ journal pages — states every finite simple group is either cyclic $\mathbb{Z}/p\mathbb{Z}$, alternating $A_n$ ($n \geq 5$), a group of Lie type, or one of 26 sporadic groups (the largest being the Monster $\mathbb{M}$, of order $\approx 8 \times 10^{53}$).
  1. Extensions and cohomology: Given $N \trianglelefteq G$ and $Q = G/N$, how many non-isomorphic $G$ realize this? The answer lives in $H^2(Q, N)$ — group cohomology — connecting group theory to algebraic topology.
  1. Profinite and $p$-adic groups: Inverse limits of finite groups yield profinite groups (e.g., $\hat{\mathbb{Z}} = \prod_p \mathbb{Z}_p$), critical in the Langlands program and number theory.

Current State

WARNING: Note: this information may be dated

Active frontiers include:

  • Moonshine and the Monster: The Monstrous Moonshine conjecture (Borcherds, Fields Medal 1998) connecting $\mathbb{M}$ to modular forms has spawned umbral moonshine and connections to string theory/vertex operator algebras that remain incompletely understood.
  • Computational group theory: GAP and Magma systems enable classification of groups up to large orders; the question of gap-filling in the CFSG proof (some steps span thousands of pages) drives ongoing formalization efforts in proof assistants like Lean.
  • Groups in quantum many-body physics: Symmetry-protected topological (SPT) phases are classified by group cohomology $H^{d+1}(G, U(1))$. Recent work on active matter (e.g., Stahl et al. 2026 on Brownian ratchets) explores broken detailed balance — asking which symmetry constraints survive far from equilibrium.
  • Quantum computation via symmetry groups: The transverse-field Ising model — whose symmetry group is $\mathbb{Z}/2\mathbb{Z}$ — was recently shown (Werner 2026) polynomially equivalent to the gate model of quantum computation, with implications for analog quantum simulation complexity.

Topology✓ reviewed

Math · Geometry  ·  ~4 min read

Core Concept

Topology studies properties of spaces preserved under continuous deformations — stretching, twisting, bending — but not tearing, gluing, or self-intersection. The central objects are topological spaces: sets $X$ equipped with a topology $\mathcal{T}$, a collection of "open" subsets satisfying three axioms:

$$ \emptyset, X \in \mathcal{T}; \quad \bigcup_\alpha U_\alpha \in \mathcal{T}; \quad U_1 \cap U_2 \in \mathcal{T} $$

(arbitrary unions and finite intersections of opens are open).

[!info]

Homeomorphism: A bijection $f: X \to Y$ where both $f$ and $f^{-1}$ are continuous. Two homeomorphic spaces are topologically identical — a coffee mug and a donut are the canonical example.

The key invariants distinguishing topological spaces include homotopy type, homology groups $H_n(X)$, fundamental group $\pi_1(X)$, and higher homotopy groups $\pi_n(X)$. A homotopy between maps $f, g: X \to Y$ is a continuous $H: X \times [0,1] \to Y$ with $H(\cdot, 0) = f$ and $H(\cdot, 1) = g$ — a formal notion of "continuously deforming one map into another."

Why It Matters

Topology sits at the intersection of geometry, algebra, and analysis, exporting tools to nearly every corner of mathematics and physics.

  • Analysis: Compactness (every open cover has a finite subcover) underlies the extreme value theorem and uniform continuity results. Connectedness formalizes the intermediate value theorem.
  • Physics: Topological invariants classify phases of matter — topological insulators, Weyl semimetals, and quantum Hall states are distinguished not by local order parameters but by global topological charges (Chern numbers, winding numbers).
  • Data science: Persistent homology extracts multi-scale topological features from point clouds — voids and loops that survive across filtration parameters encode genuine structure.
  • Dynamics: Fixed-point theorems (Brouwer, Lefschetz) guarantee solutions to differential equations without constructing them explicitly.

[!tip]

Euler's polyhedron formula $V - E + F = 2$ is topology's first theorem in disguise: the Euler characteristic $\chi = V - E + F$ is a topological invariant of the sphere $S^2$, generalized by the alternating sum of Betti numbers $\chi = \sum_k (-1)^k \text{rank}\, H_k$.

Key Details

  1. Separation axioms stratify topological spaces by how well points and closed sets can be distinguished. The Hausdorff ($T_2$) condition — distinct points have disjoint neighborhoods — is the minimal assumption for most analysis. Metric spaces are always Hausdorff.
  1. Compactness vs. sequential compactness: In metric spaces these coincide, but in general topological spaces they diverge. The correct general notion (via nets or filters) is essential in functional analysis and model theory.
  1. The fundamental group $\pi_1(X, x_0)$: Loops based at $x_0$ modulo homotopy, with concatenation as the group operation. Non-abelian in general; $\pi_1(S^1) \cong \mathbb{Z}$, $\pi_1(T^2) \cong \mathbb{Z}^2$. Higher $\pi_n$ for $n \geq 2$ are always abelian (Eckmann–Hilton argument).
  1. CW complexes provide a combinatorial handle on topology: spaces built by attaching $n$-cells $D^n$ along their boundaries $S^{n-1}$. Cellular homology computes $H_n$ efficiently from the attaching maps.
  1. Fiber bundles and characteristic classes: A bundle $F \hookrightarrow E \to B$ is locally $B \times F$ but can be globally twisted. Characteristic classes (Chern, Stiefel-Whitney, Pontryagin) in cohomology measure this twist — they are the bridge to physics via gauge theory and index theorems.

Current State

[!warning]

Note: this information may be dated

Active frontiers span several distinct regimes:

  • Symplectic and contact topology: The study of Reeb dynamics on contact manifolds $(M, \xi)$ is flourishing. Recent work (Abreu & Macarini, 2026) establishes sharp lower bounds $r_M$ for the number of simple closed Reeb orbits on manifolds with periodic equivariant symplectic homology, showing $r_M = \dim H_*(M/S^1; \mathbb{Q})$ for prequantization spaces — a striking bridge between dynamics and algebraic topology.
  • Geometric group theory: The coarse geometry of homeomorphism groups of Stone spaces (Domat, Hoganson, Lyman, 2026) gives one of the first complete coarse-equivalence classifications for non-locally compact groups, connecting to Hamming cube geometry in the intermediate class.
  • Topological phases in condensed matter: Nontrivial topology co-existing with flat bands near the Fermi level — observed in bulk TaRhTe₄ — opens platforms where topological invariants interact with strongly correlated physics outside DFT predictions.
  • $\infty$-categories and homotopy type theory: The reformulation of homotopy theory via $(\infty,1)$-categories (Lurie's work) and Voevodsky's univalence axiom in HoTT recast topology as a foundation for all mathematics.
