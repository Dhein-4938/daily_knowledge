Homotopy Type Theory✓ reviewed

Logic · Foundations  ·  ~4 min read

Core Concept

Homotopy Type Theory (HoTT) is a foundational framework for mathematics that interprets types not merely as sets or propositions, but as spaces in the sense of homotopy theory — with elements as points, equalities as paths, and higher equalities as homotopies between paths.

INFO
The central insight: in Martin-Löf dependent type theory, the identity type $\text{Id}_A(a, b)$ (proofs that $a = b$ in type $A$) is reinterpreted as the path space between points $a$ and $b$ in a topological space. This transforms equality from a flat, propositional relation into a structured, higher-dimensional object.

The key axiom distinguishing HoTT from standard type theory is the Univalence Axiom (due to Voevodsky):

$$\text{Univalence}: (A \simeq B) \simeq (A =_{\mathcal{U}} B)$$

That is, an equivalence of types $A \simeq B$ is itself equivalent to a path between $A$ and $B$ in the universe $\mathcal{U}$. Equivalences become the same thing as identifications.

Higher Inductive Types (HITs) extend this: you can define a type by specifying not just its point constructors but also its path constructors. The circle $S^1$, for instance, is the HIT with one point base : S^1 and one path loop : base = base.

Why It Matters

HoTT sits at a remarkable intersection:

  • Foundations of mathematics: It provides an alternative to ZFC set theory where homotopy-invariant structure is native, not encoded. Mathematical objects behave correctly under equivalence by construction.
  • Proof assistants: HoTT has been formalized in Agda, Coq, and Lean. Proofs are programs; the univalence axiom makes transport along equivalences automatic rather than tedious.
  • ∞-category theory: HoTT serves as the internal language of (∞,1)-toposes. Results proved synthetically in HoTT hold in any such topos simultaneously.
  • Computer science: Dependent types with univalence give a principled account of abstraction boundaries — isomorphic implementations are provably interchangeable.
TIP
Univalence enforces "representation independence" for free. If two data structures are provably equivalent, any theorem about one transfers to the other via transport. This is the formal analogue of the informal mathematical practice of treating isomorphic objects as identical.

Key Details

  1. $n$-types and truncation: Types are stratified by homotopy complexity. A proposition (mere $(-1)$-type) has at most one path between any two points. A set (0-type) has contractible path spaces. Higher $n$-types carry non-trivial homotopy up to level $n$. The truncation operator $\|{-}\|_n$ projects any type down.
  1. Path induction: The eliminator for identity types — J — says that to prove a property of all paths, it suffices to prove it for reflexivity. This is exactly the statement that the path space $\text{Id}_A(a,-)$ is contractible (has a as a center of contraction).
  1. Cubical type theory: Classical HoTT with the Univalence Axiom is not computationally satisfying — transport along univalence has no canonical reduction. Cubical type theory (CCHM, XTT, Cartesian cubical) solves this by building paths from first principles using an interval type, making univalence provable and computational.
  1. Synthetic homotopy theory: HoTT lets you calculate homotopy groups of spaces purely type-theoretically. The classical result $\pi_1(S^1) \cong \mathbb{Z}$ has a fully formal proof in HoTT requiring no point-set topology.
  1. Propositional vs. definitional equality: HoTT sharply distinguishes $a = b$ (a type, possibly inhabited by multiple paths) from $a \equiv b$ (definitional/judgmental equality, a metatheoretic assertion). Conflating these is a common source of confusion.

Current State

WARNING
Note: this information may be dated

Active research fronts include: completing Agda's cubical library and the HoTT-relevant parts of Lean 4's Mathlib; formalizing condensed mathematics and synthetic algebraic geometry in cohesive HoTT; and resolving the coherence problem for semi-simplicial types (defining Kan complexes internally without assuming univalence). The HoTT Book (IAS, 2013) remains the canonical reference, but cubical developments have substantially moved the computational story forward.

Discussion Prompts

  1. Univalence makes equivalence and equality interchangeable — but does this eliminate the distinction between "same structure" and "same object," or merely relocate the philosophical tension?
  2. Cubical type theory restores computational content to univalence, but at the cost of a more complex type theory. Is there a fundamental trade-off between synthetic elegance and computational tractability in foundations?
  3. If HoTT is the internal language of ∞-toposes, what would it mean to do physics — e.g., gauge theory — entirely synthetically inside a suitable topos, without ever mentioning manifolds or fiber bundles directly?
