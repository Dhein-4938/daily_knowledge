Game Semantics✓ reviewed

Logic · Semantics  ·  ~4 min read

Core Concept

Game semantics reinterprets logical truth not as static set-theoretic satisfaction, but as the existence of a winning strategy in an adversarial dialogue game. A formula $\varphi$ is "true" or "valid" iff a designated player — the Proponent (P) or Verifier — has a winning strategy against the Opponent (O) or Falsifier, regardless of O's moves.

INFO
Formal kernel. Given formula $\varphi$, the game $\mathcal{G}(\varphi)$ is defined inductively on the structure of $\varphi$: - $\varphi \land \psi$: O chooses which conjunct to play next. - $\varphi \lor \psi$: P chooses. - $\neg \varphi$: roles swap; play continues in $\mathcal{G}(\varphi)$. - $\exists x.\, \varphi(x)$: P provides a witness $t$; play continues in $\mathcal{G}(\varphi(t))$. - $\forall x.\, \varphi(x)$: O chooses the witness. P wins an atomic position iff the atom is true in the underlying model.

A winning strategy for P is a function from game histories to P-moves such that every play consistent with the strategy ends in a P-win. $\varphi$ is valid iff such a strategy exists. This immediately connects semantic truth to constructive content: a proof of $\exists x.\,\varphi(x)$ must exhibit the witness.

Two major traditions diverge here:

  • Dialogical Logic (Lorenzen & Lorenz, 1950s–60s): rules are purely syntactic; no background model. Validity = P has a formal winning strategy.
  • Game-Theoretical Semantics / GTS (Hintikka, 1960s–): a model $\mathcal{M}$ is fixed; games are evaluation games over $\mathcal{M}$.

Why It Matters

Game semantics is more than a curiosity — it fills genuine gaps left by Tarskian model theory.

  1. Intuitionistic and substructural logics. The Curry-Howard correspondence tells us proofs are programs, but what are proofs of? Game semantics gives a direct answer: a proof is a P-strategy. This is especially clean for linear logic (Girard), where structural rules (contraction, weakening) correspond precisely to game resources.
  1. Full abstraction for programming languages. Abramsky, Jagadeesan, Malacaria, and Hyland & Ong (mid-1990s) used game semantics to solve the long-open full abstraction problem for PCF (a typed $\lambda$-calculus with recursion). A denotational model is fully abstract iff contextual equivalence ⟺ semantic equality; game models achieve this where Scott domains famously failed.
  1. Concurrency and interaction. Concurrent computation is inherently dialogic. Game models naturally handle the interleaving of moves between components, making them a natural fit for process calculi and session types.
TIP
Game semantics makes the compositionality of interaction structurally explicit: composing two strategies (via "hiding" internal moves) corresponds directly to cut-elimination in proof theory and to linking components in a hardware or software system.

Key Details

  • Strategies as morphisms. In the Abramsky–Hyland categorical framework, arenas (game types) are objects and strategies (P's behavior) are morphisms. Composition is defined by parallel play plus hiding — a precise categorical structure (compact closed, or *-autonomous categories for linear logic).
  • Innocence and history-freeness. A strategy is innocent if P's move depends only on the current O-move and P's own prior moves, not O's full history. Innocence characterizes definability in higher-order functional languages.
  • Positionality vs. history-dependence. Classical logic admits positional (memoryless) winning strategies for certain games; intuitionistic logic typically requires history-sensitivity, reflecting the constructive content of proofs.
  • Independence-Friendly (IF) logic. Hintikka extended GTS to IF logic, where $\forall / \exists$ quantifiers can be informationally independent — P cannot see all of O's prior choices. This captures imperfect information and yields a logic strictly beyond first-order expressibility.
  • Complexity classes via games. Polynomial-time strategies in evaluation games characterize complexity classes: e.g., PTIME ↔ strategies computable in polynomial time. This gives a game-semantic account of descriptive complexity.

Current State

WARNING
Note: this information may be dated

Active threads as of the mid-2020s include:

  • Higher-order model checking using game semantics (Ong's HORS model-checking, decidability results).
  • Probabilistic game semantics: extending arenas and strategies to handle probabilistic branching, relevant to probabilistic programming languages.
  • Quantum game semantics: replacing deterministic strategies with quantum strategies (superposition of moves), connecting to categorical quantum mechanics.
  • Session types and deadlock freedom: game-theoretic proofs of type-safety properties in concurrent $\pi$-calculi.
  • Automated synthesis: game-solving algorithms applied directly to program synthesis and reactive system verification.

Discussion Prompts

  1. Dialogical logic derives validity without a background model, while GTS fixes one — does this difference matter for foundations, or is it merely a presentation choice? Can one be faithfully embedded in the other?
  1. Innocence restricts P-strategies to those that ignore O's history; what is the precise correspondence between innocence and purely functional (side-effect-free) computation, and how does it break down for stateful languages?
  1. IF logic escapes the Löwenheim-Skolem theorem and collapses the $\Sigma^1_1 / \Pi^1_1$ boundary in expressiveness — does this make it a better logic for mathematics, or do the costs (loss of compactness, non-axiomatizability) outweigh the gains?
