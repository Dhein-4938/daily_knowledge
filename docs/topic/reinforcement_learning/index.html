Reinforcement Learning✓ reviewed

Tech · Machine Learning  ·  ~4 min read

Core Concept

INFO: Definition
Reinforcement Learning (RL) is the study of how an agent learns a policy $\pi: \mathcal{S} \to \mathcal{A}$ by interacting with an environment to maximize cumulative reward. Unlike supervised learning, no labeled input-output pairs are provided — feedback is scalar, delayed, and often sparse.

The canonical formalism is the Markov Decision Process (MDP), defined by the tuple $(\mathcal{S}, \mathcal{A}, P, R, \gamma)$:

$$G_t = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1}, \quad \gamma \in [0, 1)$$

The agent seeks a policy $\pi^*$ that maximizes the expected return $\mathbb{E}_\pi[G_t]$. The Bellman optimality equation captures the recursive structure:

$$Q^*(s, a) = \mathbb{E}\left[R_{t+1} + \gamma \max_{a'} Q^*(s', a') \mid s, a\right]$$

Why It Matters

RL is the closest computational analogue to how biological organisms learn from consequences. It underpins:

  • Game-playing AI (AlphaGo, AlphaZero, OpenAI Five)
  • Robotics — sim-to-real transfer for dexterous manipulation
  • LLM alignment — RLHF and GRPO are RL at their core
  • Operations research — scheduling, inventory control, routing
  • Neuroscience — dopamine circuits mirror temporal difference (TD) learning
TIP: The RL–Optimal Control Bridge
Classical optimal control (e.g., LQR, MPC) assumes a known dynamics model $P$. RL relaxes this: it learns from sampled transitions. This makes RL applicable wherever simulation or real interaction is cheaper than deriving an analytical model — which is most of the interesting world.

Key Details

1. Exploration–Exploitation Dilemma

The agent must balance exploiting known-good actions vs. exploring to reduce uncertainty. Formal treatments include UCB (Upper Confidence Bound), Thompson Sampling, and the $\epsilon$-greedy heuristic. In continuous/deep RL, this remains largely heuristic.

2. Value-Based vs. Policy Gradient Methods

  • Value-based (DQN, Q-learning): learn $Q^\pi$ implicitly, derive policy greedily. Sample-efficient but biased in continuous action spaces.
  • Policy gradient (REINFORCE, PPO, GRPO): directly optimize $\nabla_\theta \mathbb{E}[G_t]$. Unbiased but high-variance. Actor-critic methods blend both.

3. Credit Assignment Problem

Rewards are often delayed by hundreds of steps. Attributing which earlier actions caused a later reward is non-trivial — temporal difference learning addresses this by bootstrapping from value estimates rather than waiting for episode termination.

4. The Deadly Triad

Combining function approximation + bootstrapping + off-policy learning can cause instability or divergence. DQN circumvents this with experience replay and target networks; other approaches (e.g., distributional RL, constrained updates in PPO) add further stabilizers.

5. Reward Hacking

An agent optimizing a proxy reward $\hat{R}$ can find unintended solutions that score well on $\hat{R}$ while violating the true objective. This is Goodhart's Law in action and is a central concern in AI alignment.

Current State

WARNING: Note: this information may be dated

Active frontiers as of mid-2026:

  • RL Fine-Tuning of LLMs: GRPO (Group Relative Policy Optimization) has displaced PPO in many LLM post-training pipelines due to lower memory overhead. Papers like RA-RFT extend this with reasoning-aware retrieval, boosting AIME 2025 accuracy by 7+ points over vanilla GRPO.
  • Agentic / Multi-Step RL: InterleaveThinker applies step-wise GRPO rewards across 25+ generator calls in a trajectory — highlighting that RL signal need not be end-to-end to guide long-horizon tasks.
  • Sim-to-Real for Dexterous Robots: Mana demonstrates zero-shot transfer for articulated tool manipulation via coarse-to-fine RL pipelines, reducing human annotation to under a minute per tool.
  • Flow Matching + RL: FRS shows that flow-matching generalist policies can be steered via latent-space reversal, bootstrapping RL with semantic priors where standard RL fails entirely.
  • World Action Models: Representation-centric tokenization (RepWAM) is emerging as a way to jointly model future visual states and latent actions — potentially unifying prediction and control.

Discussion Prompts

  1. Reward specification vs. reward learning: Given that hand-crafted rewards reliably produce Goodhart pathologies, how far can inverse RL or preference-based methods take us before they introduce their own alignment failure modes?
  1. Is the MDP assumption load-bearing? Many real problems violate the Markov property (partial observability, non-stationarity). How much does the POMDP generalization actually help in practice, and when is it just theoretical machinery?
  1. RL for LLMs as a paradigm shift: GRPO treats token generation as a policy and outcome correctness as a verifiable reward — essentially collapsing the distinction between "reasoning" and "control." What does this imply for how we should think about intelligence architectures going forward?
