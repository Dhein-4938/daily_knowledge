Lambda Calculus✓ reviewed

Logic · Computation Theory  ·  ~4 min read

Core Concept

Lambda calculus (λ-calculus) is a formal system for expressing computation through function abstraction and function application. Introduced by Alonzo Church in the 1930s, it is one of the two foundational models of computation — the other being the Turing Machine — and they are provably equivalent in expressive power.

INFO: Syntax (BNF)
Every λ-term $e$ is one of: $$e ::= x \mid \lambda x.\, e \mid e_1\; e_2$$ — a variable, a lambda abstraction (binding $x$ in body $e$), or an application (applying $e_1$ to $e_2$).

The sole computational rule is β-reduction:

$$(\lambda x.\, e_1)\; e_2 \;\longrightarrow_\beta\; e_1[x := e_2]$$

Substituting $e_2$ for free occurrences of $x$ in $e_1$. An additional rule, α-conversion, allows bound variable renaming to avoid capture:

$$\lambda x.\, e \;\equiv_\alpha\; \lambda y.\, e[x := y] \quad (y \notin \mathrm{FV}(e))$$

η-conversion captures extensionality: $\lambda x.\, f\, x \equiv_\eta f$ when $x \notin \mathrm{FV}(f)$.

Why It Matters

λ-calculus is the theoretical backbone of functional programming — Haskell, ML, Lisp, and Scheme are all direct descendants. Its significance extends far beyond PL design:

  • Foundations of mathematics: Church used it to investigate decidability, leading directly to the negative solution of the Entscheidungsproblem (independently resolved by Turing).
  • Proof theory: The Curry-Howard Correspondence maps λ-terms to proofs and types to propositions — programs are proofs.
  • Denotational semantics: Provides the mathematical substrate for giving meaning to programs without executing them.
  • Type theory and dependent types: Typed λ-calculi (simply-typed, System F, CoC) underpin proof assistants like Coq and Lean.
TIP: Church–Turing Thesis connection
The equivalence of untyped λ-calculus and Turing machines is non-trivial: λ-calculus encodes data (via Church numerals), conditionals, and recursion (via the Y combinator) purely through functions. There are no side effects, no memory — only substitution. This makes it an unusually pure model of computation.

Key Details

1. Church Encodings — Natural numbers, booleans, and pairs can all be represented as pure functions. The Church numeral for $n$ applies a function $f$ exactly $n$ times:

$$\bar{n} \equiv \lambda f.\,\lambda x.\, f^n(x)$$

2. The Y Combinator — Recursion without named functions is achieved via the fixed-point combinator:

$$Y \equiv \lambda f.\,(\lambda x.\, f\,(x\,x))\,(\lambda x.\, f\,(x\,x))$$

$Y\,g$ reduces to $g\,(Y\,g)$, threading recursion through self-application. Note: $Y$ has no normal form in the untyped calculus — it diverges if $g$ doesn't force termination.

3. Confluence (Church-Rosser Theorem) — If a term $e$ reduces to both $e_1$ and $e_2$, there exists a $e_3$ such that both reduce to $e_3$. This guarantees determinism of normal forms: evaluation order affects efficiency, not result.

4. Undecidability of β-equality — There is no algorithm to decide whether two arbitrary λ-terms are β-equal. This is the λ-calculus analogue of the halting problem.

5. Typed vs. Untyped — Untyped λ-calculus is Turing-complete but permits non-termination. Simply-typed λ-calculus (STLC) is strongly normalizing (all reductions terminate) but not Turing-complete — a fundamental trade-off between expressivity and decidability.

Current State

WARNING: Note: this information may be dated

Active research frontiers include:

  • Homotopy Type Theory (HoTT): Extensions of dependent type theory (rooted in typed λ-calculi) that treat type equality as path spaces from Homotopy Theory.
  • Probabilistic and quantum λ-calculi: Extending the formalism to model randomized and quantum computation.
  • Optimal reduction: Lévy's work on sharing redexes (implemented in interaction nets) seeks the minimal number of β-steps — still an open efficiency problem for practical implementations.
  • Linear λ-calculus: Restricting variable use to exactly once, connecting to Linear Logic and resource-sensitive computation models.

Discussion Prompts

  1. The Y combinator self-applies a term to itself — in what precise sense does this "simulate" recursion, and why does the simply-typed λ-calculus reject it?
  2. Church-Rosser guarantees confluence of normal forms, yet evaluation order matters operationally (call-by-name vs. call-by-value). How do these two facts coexist without contradiction?
  3. If untyped λ-calculus is Turing-complete and dependent type theory is not, what does this imply about the relationship between provability and computability?
