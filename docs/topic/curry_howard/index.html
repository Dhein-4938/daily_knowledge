Curry-Howard Correspondence✓ reviewed

Logic · Foundations  ·  ~4 min read

Core Concept

INFO: Definition
The Curry–Howard correspondence (also isomorphism or proofs-as-programs) is a structural identity between systems of formal logic and models of computation: propositions are types, and proofs are programs.

The correspondence is not merely metaphorical — it is an exact syntactic and semantic isomorphism between derivation trees in a logic and well-typed terms in a lambda calculus. The canonical alignment:

| Logic | Type Theory |

|---|---|

| Proposition $P$ | Type $\tau$ |

| Proof of $P$ | Term $t : \tau$ |

| Implication $P \Rightarrow Q$ | Function type $\tau \to \sigma$ |

| Conjunction $P \land Q$ | Product type $\tau \times \sigma$ |

| Disjunction $P \lor Q$ | Sum type $\tau + \sigma$ |

| Falsehood $\bot$ | Empty type $\mathbf{0}$ |

| Universal quantification $\forall x.\ P(x)$ | Dependent product $\Pi (x : A).\ B(x)$ |

| Existential quantification $\exists x.\ P(x)$ | Dependent sum $\Sigma (x : A).\ B(x)$ |

The base case: intuitionistic propositional logic corresponds exactly to the simply typed lambda calculus ($\lambda^\to$). Extend to first-order logic and you arrive at dependent type theory — the foundation of proof assistants like Coq and Agda.

$$ \frac{\Gamma, x : \tau \vdash t : \sigma}{\Gamma \vdash \lambda x.\ t : \tau \to \sigma} \quad\longleftrightarrow\quad \frac{\Gamma, P \vdash \pi : Q}{\Gamma \vdash \lambda\text{-abstraction} : P \Rightarrow Q} $$

Proof normalization (cut-elimination) corresponds exactly to beta-reduction in the lambda calculus — computation is proof simplification.

Why It Matters

This is not a curiosity — it is the deepest bridge between mathematics and computer science:

  • Proof assistants: Coq, Agda, Lean, and Idris are direct engineering consequences. Writing a type-correct program is constructing a proof.
  • Certified software: Security protocols, compilers (CompCert), and OS kernels can be verified by encoding correctness as a type that the program must inhabit.
  • Type system design: Linear types correspond to linear logic; session types correspond to classical sequent calculi; effect systems connect to modal logics.
  • Foundations of mathematics: Homotopy Type Theory (HoTT) extends the correspondence into homotopy theory, where types are spaces and proofs are paths.
TIP: The Constructive Constraint
The correspondence holds for intuitionistic (constructive) logic, not classical logic. The law of excluded middle ($P \lor \neg P$) has no direct computational content — its "proof" doesn't tell you which side holds. Classical logic corresponds instead to lambda calculi with continuations (Parigot's $\lambda\mu$-calculus), where $\neg\neg P \Rightarrow P$ maps to double-negation elimination via callcc.

Key Details

  1. BHK Interpretation precedes it: Brouwer–Heyting–Kolmogorov gave the semantic reading (a proof of $P \Rightarrow Q$ is a function from proofs of $P$ to proofs of $Q$); Curry–Howard made this syntactic and formal.
  1. Normalization = Termination: Strong normalization of the typed calculus corresponds to consistency of the logic (no proof of $\bot$). An inconsistent type system allows non-terminating programs to inhabit every type — undefined :: a in Haskell being the canonical cheat.
  1. The Lambek extension: Category theory completes the trinity. A cartesian closed category (CCC) simultaneously models both the typed lambda calculus and intuitionistic logic. Objects = types/propositions; morphisms = terms/proofs; composition = cut/function application.
  1. Linear logic correspondence: Girard's linear logic (1987) corresponds to session types and resource-aware computation — a "use exactly once" proposition maps to a linear function type.
  1. Proof relevance: In dependent type theory, the identity of two proofs of the same proposition can matter (proof relevance). HoTT radicalizes this: two proofs being "equal" becomes a homotopy path between points in a space.
WARNING: Note: this information may be dated
Active research areas like cubical type theory implementations, the precise computational content of classical logic extensions, and HoTT's homotopy-theoretic foundations continue to evolve rapidly.

Current State

The correspondence has expanded well beyond its origins:

  • Cubical type theory (Cohen et al., 2016 onward) gives computational content to the univalence axiom in HoTT, resolving a long-standing gap where the axiom had no canonical reduction rule.
  • Linear and graded type systems (e.g., Granule) track resource usage via correspondences with substructural logics.
  • Realizability theory generalizes the correspondence beyond lambda calculi to arbitrary models of computation (Turing machines, PCAs).
  • Proof mining uses the correspondence in reverse: extract computational witnesses (algorithms, bounds) from classical proofs by translating them into constructive form.

Discussion Prompts

  1. If classical logic corresponds to continuations, does that mean every classical proof has a "computational cost" payable in control-flow complexity — and can this cost be made precise?
  1. Proof normalization and program evaluation are the same process under the isomorphism — so what does proof complexity (the study of how long proofs can be) say about computational complexity classes like P vs. NP?
  1. HoTT treats proofs of $a = b$ as paths in a space, making proof identity geometrically meaningful. Does this suggest that the "intensional" vs. "extensional" type theory debate is really a question about the topology of the proof space?
