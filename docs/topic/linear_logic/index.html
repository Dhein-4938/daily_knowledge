Linear Logic✓ reviewed

Logic · Logic  ·  ~5 min read

Core Concept

INFO: Definition
Linear logic is a substructural logic introduced by Jean-Yves Girard (1987) that controls the use of the structural rules of contraction (using a hypothesis multiple times) and weakening (discarding a hypothesis). The result is a logic where propositions denote resources — entities that must be used exactly once unless explicitly marked otherwise.

In classical sequent calculus, Gentzen's structural rules allow:

$$\frac{\Gamma, A, A \vdash B}{\Gamma, A \vdash B} \text{(contraction)} \qquad \frac{\Gamma \vdash B}{\Gamma, A \vdash B} \text{(weakening)}$$

Linear logic restricts both. The canonical reading: if $A \vdash B$, consuming resource $A$ produces resource $B$ — you cannot freely copy or discard $A$.

Linear logic introduces a richer connective vocabulary to recover expressiveness:

| Connective | Symbol | Flavor | Intuition |

|---|---|---|---|

| Tensor | $A \otimes B$ | Multiplicative | Have both A and B simultaneously |

| Par | $A \parr B$ | Multiplicative | Dual of tensor |

| With | $A \mathbin{\&} B$ | Additive | Choose one of A or B |

| Plus | $A \oplus B$ | Additive | Externally given one of A or B |

| Of course | $!A$ | Exponential | A is unlimited (contraction/weakening restored) |

| Why not | $?A$ | Exponential | Dual of $!A$ |

The exponential modalities $!$ and $?$ are the precise gatekeepers that re-admit classical structural freedom when needed.

Why It Matters

Linear logic sits at an unusual crossroads — it is simultaneously a foundational proof theory, a practical programming language model, and a physics-adjacent formalism.

Programming languages. The Curry–Howard correspondence extends to linear logic: linear types correspond to linear proofs. A linear type system enforces that a value is used exactly once, making it ideal for modeling:

  • Memory ownership (Rust's borrow checker is a practical descendant)
  • Session types for typed communication protocols
  • Safe resource management without garbage collection

Game semantics. Girard's Geometry of Interaction reframes cut-elimination as token passing on a graph, with proofs as strategies in a game between Proponent and Opponent. This gives a fully abstract model for PCF and related languages.

Quantum information. No-cloning and no-deleting theorems in quantum mechanics mirror exactly the prohibition on contraction and weakening. Linear logic is therefore a natural logical substrate for quantum computation — $!A$ corresponds to classical (freely copyable) data.

TIP: The Resource Metaphor
Think of linear implication $A \multimap B$ as a vending machine: insert one $A$, receive one $B$. The machine is consumed in the transaction. Classical implication $A \Rightarrow B$ is a theorem that can be applied infinitely; linear implication is a process.

Key Details

  1. Multiplicative vs. Additive split. The two flavors of conjunction ($\otimes$ vs $\&$) and disjunction ($\parr$ vs $\oplus$) are genuinely distinct. $A \otimes B$ means you hold both resources simultaneously; $A \mathbin{\&} B$ means you can redeem either, but choosing one forfeits the other. Confusing these is the most common error for newcomers.
  1. MLL is decidable; MELL is not. Multiplicative linear logic (MLL, without exponentials) is decidable and has clean combinatorial proof theory. Adding exponentials ($!$/$?$) yields MELL, which is Turing-complete and undecidable — the exponentials recover the full power of classical logic.
  1. Proof nets. Girard introduced proof nets as a canonical (syntax-independent) representation of MLL proofs, eliminating bureaucratic rule-permutations. Correctness of a proof net is checkable by the Danos–Regnier criterion (acyclicity + connectedness of a "correction graph"). Extending proof nets to MELL remains an active research problem.
  1. Denotational semantics via monoidal categories. Linear logic corresponds to -autonomous categories (the categorical semantics of classical linear logic), while intuitionistic linear logic corresponds to symmetric monoidal closed categories. This is a strict refinement: cartesian closed categories (for intuitionistic logic) embed into SMCCs via the $!$ comonad.
  1. Phase semantics. The original Girard semantics uses phases — a commutative monoid with a distinguished set of "facts." This gives completeness but is not fully abstract; coherence spaces and later game-semantic models provide more refined interpretations.

Current State

WARNING: Note: this information may be dated

Active research threads include:

  • Differential linear logic (Ehrhard & Regnier): adds a differentiation operator, connecting linear logic to Taylor expansion and the semantics of probabilistic and differentiable programs.
  • Graded/quantitative type systems: generalizing $!$ to a semiring of usage counts (e.g., use at most twice), giving finer resource tracking. This underlies systems like Granule and Idris 2's quantitative types.
  • Cyclic and non-wellfounded proofs: extending linear logic to infinite proof trees with productivity conditions, relevant for reasoning about coinductive data and streams.
  • Linear logic and deep learning: the connection to differential linear logic and automatic differentiation is being explored as a semantic foundation for gradient-based learning.

Discussion Prompts

  1. The $!$ modality restores contraction and weakening — so classical logic embeds into linear logic via $!A \vdash B \Leftrightarrow A \Rightarrow B$. Does this mean linear logic is strictly more expressive than classical logic, or merely a finer decomposition of the same truths? What does "expressive" even mean here?
  1. Rust's ownership system is often cited as a practical descendant of linear types, yet it adds borrowing (temporary shared access) which has no clean linear logic counterpart. What is the right logical primitive for borrowing, and does it break the resource metaphor?
  1. The no-cloning theorem makes linear logic appealing for quantum computation, but quantum states live in Hilbert spaces with superposition — a phenomenon classical linear logic doesn't model. What extensions (e.g., probabilistic linear logic, quantum lambda calculi) are necessary, and do they preserve the clean proof-theoretic properties of the original?
