Computability Theory✓ reviewed

Logic · Computation Theory  ·  ~5 min read

Core Concept

Computability theory (also recursion theory) is the branch of mathematical logic and theoretical computer science concerned with formalizing what it means for a function to be effectively calculable, and with classifying functions that are not.

INFO: Church-Turing Thesis
Every effectively calculable function is computable by a Turing machine. This is not a theorem — it's a definition-level thesis that identifies the informal notion of "algorithm" with a precise mathematical model. The major equivalent models (λ-calculus, general recursive functions, Turing machines) all converge on the same class.

The primary object of study is the class of partial computable (recursive) functions $f : \mathbb{N}^k \rightharpoonup \mathbb{N}$, built from zero, successor, and projection via composition, primitive recursion, and the minimization ($\mu$) operator. Functions requiring $\mu$ may diverge — they are partial, not total.

A set $A \subseteq \mathbb{N}$ is computably enumerable (c.e.) if it is the domain of some partial computable function, equivalently if a Turing machine can enumerate its members (possibly without halting on non-members). $A$ is computable iff both $A$ and $\bar{A}$ are c.e.

The canonical non-computable problem is the Halting Problem $K = \{e \mid \varphi_e(e) \downarrow\}$, which is c.e. but not computable. Its undecidability is proven by diagonalization: assume a decider exists, construct a contradiction.

Why It Matters

Computability theory draws the hard boundary of what any algorithm can ever do — not due to engineering limits, but in principle. Results here are absolute: no faster hardware, no cleverer language, no quantum speedup crosses the boundary set by Rice's Theorem or the undecidability of first-order logic.

Connections fan out widely:

  • Proof theory: Gödel's incompleteness theorems are essentially computability arguments in disguise — unprovable sentences correspond to non-computable predicates.
  • Descriptive set theory: The arithmetical hierarchy $\Sigma^0_n / \Pi^0_n$ mirrors the Borel hierarchy of sets of reals; effective versions (lightface) are central to both fields.
  • Complexity theory: Computability theory is the "infinite-resource" ceiling above which complexity theory carves out finer, resource-bounded distinctions like $\mathsf{P}$ vs $\mathsf{NP}$.
  • Formal verification: Undecidability results (e.g., program equivalence, termination) set hard limits on what static analysis and model checkers can guarantee.

Key Details

  1. The Arithmetical Hierarchy. Sets are stratified by quantifier alternation over $\mathbb{N}$:

$$A \in \Sigma^0_n \iff A = \{x \mid \exists y_1\, \forall y_2\, \cdots\, Q y_n\; R(x, \vec{y})\}$$

$\Sigma^0_1$ = c.e. sets; $\Delta^0_1$ = computable sets. The hierarchy is strict and does not collapse.

  1. Turing Reducibility and Degrees. $A \leq_T B$ (A is Turing-reducible to B) if a Turing machine with oracle $B$ can decide $A$. This induces an equivalence relation; the equivalence classes are Turing degrees. The degrees form an upper semilattice under join $A \oplus B$, but lack a meet in general — the structure is extraordinarily complex (every countable partial order embeds into it).
  1. The Jump Operator. $A' = \{e \mid \varphi_e^A(e) \downarrow\}$ (the Turing jump) strictly increases Turing degree: $A <_T A'$ always. Iterating gives $\mathbf{0}^{(n)}$, the degree of $\Sigma^0_n$-complete sets.
  1. Rice's Theorem. For any non-trivial semantic property $P$ of partial computable functions, the index set $\{e \mid \varphi_e \in P\}$ is undecidable. "Non-trivial" means $P$ holds for some functions and fails for others. This kills most hopes of static analysis in one stroke.
  1. Post's Problem and the Priority Method. Emil Post asked (1944) whether there exist c.e. degrees strictly between $\mathbf{0}$ and $\mathbf{0}'$. Friedberg and Muchnik (1956) independently answered yes, using the finite injury priority argument — a foundational technique for constructing degrees with incompatibility constraints.
TIP: Recursion ≠ Repetition
The term "recursion theory" predates the programming usage of "recursion." It refers to general recursive functions (Gödel-Herbrand-Kleene), not to self-referential procedure calls. The renaming to "computability theory" reflects a cleaner conceptual framing, not a change of subject.

Current State

WARNING: Note: this information may be dated

Active research threads include:

  • Reverse mathematics: Calibrating which set-existence axioms are needed to prove classical theorems, using computable mathematics as the baseline ($\mathsf{RCA}_0$). The "Big Five" subsystems reveal surprising logical distinctions within ordinary mathematics.
  • Computable structure theory: Which countable algebraic/topological structures have computable presentations? What is the computable categoricity spectrum? (E.g., linear orders, Boolean algebras, groups.)
  • Continuous and higher computability: Extending the framework to real-number computation (Type-2 theory, computable analysis) and higher-type functionals — bridging to constructive analysis and homotopy type theory.
  • Algorithmic randomness: Characterizing random sequences via incompressibility (Kolmogorov complexity) or unpredictability (martingales). Martin-Löf randomness, Schnorr randomness, and their degree-theoretic properties remain actively studied.

Discussion Prompts

  1. Rice's Theorem rules out deciding any non-trivial semantic property of programs — but real-world static analyzers clearly do something useful. What is the precise nature of the approximation they make, and where exactly does undecidability bite in practice?
  1. The Turing degree structure is known to be extremely wild (every countable partial order embeds in the c.e. degrees). Does this structural complexity reflect something deep about the nature of relative information, or is it a pathology of an overly coarse equivalence relation?
  1. Computable analysis requires computable real numbers to be given as convergent sequences of rationals with a computable modulus of convergence — yet most reals are not computable. What does it mean to "compute" with an object you cannot even name, and how does this tension resolve in practice (e.g., in verified numerical algorithms)?
